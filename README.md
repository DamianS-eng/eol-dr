# EOL DR / End-of-life Disaster Response

This is a personal fork thanks to the efforts made by another repository holder who has endured a lot of the hardships of figuring out end of life plans with little to no guidance or proper information. My plan is to document much of how this home server here works.

The entire checklist needed is styled here in the markdown document, with a conversion to a Microsoft Word document as kind of a legacy format for those who need it.

# checklist.md -> checklist.docx

The plan is to write , with explicit details such as passwords to be handwritten and stored offline in a fireproof bag, **not** in a safe. Somewhere accessible, but protected from disaster. 

So here is the checklist:

* In markdown format: [checklist.md](/checklist.md)
* In docx format, generated by the GitHub Action: [checklist.docx](/checklist.docx)
